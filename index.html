游戏中心服

  • v1.0 林益远 2019.2.15 创建

1 前言

1.1 项目说明

该项目使用laravel5.6 进行开发，构建一个游戏 中心服接口

1.2 注意事项

主要的业务有以下:

  • 登录模块
  • 支付模块
  • 夺宝赛模块
  • 抽奖模块
  • …..

2 如何部署

2.1 开发说明

  • 开发框架：Laravel 5.6
  • PHP版本：PHP 7.3
  • 扩展：Redis、Swoole

2.2 安装

2.2.1 基本要求

  • 服务器要求：
    • PHP >= 7.1.3
    • OpenSSL PHP扩展
    • PDO PHP扩展，注意需要php_mysql
    • Mbstring PHP扩展
    • Tokenizer PHP扩展
    • XML PHP扩展
    • Swoole PHP扩展
    • Redis PHP扩展

2.2.2 安装步骤

以下为本项目git仓库地址

git@gitee.com:linyiyuan/New-SiRuiCenter.git

**第1步：克隆代码** 
git clone git@gitee.com:linyiyuan/New-SiRuiCenter.git

**第2步：安装composer包** 
composer install

**第3步：配置文件**

1、在项目中找到.env.example文件，该文件作为项目的全局配置文件，在部署时需要复制成.env，执行以下命令

cp -f .env.example ./.env

2、根据.env文件修改各配置项，如果.env文件中没有存在key值则运行命令：

php artisan key:generate

3、配置stroage bootstrap 可写

chmod -R 777 stroage bootstrap

**第4步：初始化数据库**

1.打开 service.sh 脚本,配置脚本里的项目路径

xm_path=项目目录(绝对路径)

配置参考

xm_path=/web/New-SiRuiCenter

赋脚本执行权限

chmod 777 service.sh

2.在.env 配置文件中配置数据库链接，并且在.env配置文件中写入如下配置

//接口入口访问开关
APP_SWITCH=true

//MySQL备份还原相关配置(绝对路径)
MYSQL_BACKUP_PATH=/mysql_backup        //备份路径
MYSQL_BIN_PATH=/usr/local/mysql/bin        //mysql数据库运行文件
MYSQL_CNF_PATH=/etc/my.cnf            //mysql数据库配置

3.配置数据库配置文件 databases.php 里添加要初始化数据库的文件路径(sql文件放在 ‘/database/init/‘ 目录下)

'init' => 'base.sql', //初始化导入数据库文件

注意：配置多个数据库时，都需要在每个数据库中的配置项中添加 "init => xxx.sql" 这一行

详细配置参考:

'mysql' => [
            'read' => [
                'host' => env('DB_READ', '127.0.0.1'),
            ],

            'write' => [
                'host' => env('DB_WRITE', '127.0.0.1'),
            ],

            'driver' => 'mysql',
            'port' => env('DB_PORT', '3306'),
            'database' => env('DB_DATABASE', 'forge'),
            'username' => env('DB_USERNAME', 'forge'),
            'password' => env('DB_PASSWORD', ''),
            'unix_socket' => env('DB_SOCKET', ''),
            'charset' => 'utf8mb4',
            'collation' => 'utf8mb4_unicode_ci',
            'prefix' => env('DB_PREFIX', ''),
            'prefix_indexes' => true,
            'strict' => false,
            'engine' => null,
            'init'   => 'base.sql', //初始化导入数据库文件
        ],

4.执行部署更新

(1).停止程序服务
    ./service.sh stop
(2).更新升级
    ./service.sh update
(3).启动程序服务
    ./service.sh start

第5步：初始化队列服务

该项目使用了基于Redis实现的队列服务，在部署项目时需要启动该队列服务，具体配置表位于config/queue.php文件，在该项目中使用了horizon拓展来进行队列的管理，具体配置表位于config/horizon.php

启动队列服务

php artisan horizon

停止队列服务

php artisan horizon:terminate

生产环境中，我们需要配置一个进程管理工具来监控 php artisan horizon 命令的执行，以便在其意外退出时自动重启。当服务器部署新代码时，需要终止当前 Horizon 主进程，然后通过进程管理工具来重启，从而使用最新的代码。
